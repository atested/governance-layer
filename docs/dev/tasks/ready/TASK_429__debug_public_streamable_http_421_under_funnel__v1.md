1. PURPOSE

Identify and fix the specific HTTP 421 failure on authenticated public requests to the Funnel-backed GovMCP `/mcp` endpoint, without changing the remote auth model, deployment model, or broader remote architecture.

2. REPRODUCTION_RESULT

- Localhost authenticated MCP path succeeded before the fix:
  - authenticated MCP client call to `http://127.0.0.1:6100/mcp` reached the GovMCP surface and returned a governed `DENY`
- Public authenticated path failed before the fix:
  - unauthenticated public `GET https://mac-mini.tail341fb0.ts.net/mcp` returned `401`
  - wrong-token public `GET https://mac-mini.tail341fb0.ts.net/mcp` returned `401`
  - correct-token public `GET https://mac-mini.tail341fb0.ts.net/mcp` returned `421 Misdirected Request`
  - correct-token public MCP client call to `https://mac-mini.tail341fb0.ts.net/mcp` also failed with `421`
- The same failure was reproducible locally without Funnel by sending the public Host header to the local service before the fix.

3. ROOT_CAUSE

The 421 was generated by the MCP SDK transport layer, not by the custom bearer-auth middleware.

- The pinned FastMCP server auto-enables DNS rebinding protection when bound to `127.0.0.1`
- In that auto-configured mode it only allows localhost Host/Origin values
- The remote GovMCP app reused that default transport-security config even when started for public deployment behind Funnel
- After bearer auth succeeded, the streamable-http transport rejected `Host: mac-mini.tail341fb0.ts.net` as an invalid host and returned `421 Misdirected Request`

Actual cause:
- FastMCP transport-security host validation mismatch between localhost-only defaults and the configured public HTTPS host under Funnel

4. IMPLEMENTATION

Implemented the smallest bounded fix in the remote-serving layer:

- updated [mcp/remote_server.py](/Volumes/SSD/archive/gov/governance-layer/mcp/remote_server.py)
  - derive remote transport-security settings from `GOVMCP_PUBLIC_BASE_URL`
  - preserve localhost host/origin allowances
  - add the configured public HTTPS host/origin to the allowed transport-security set
  - rebuild the MCP streamable-http session manager with the updated transport-security settings before constructing the remote app
- added [tests/test_mcp_remote_public_host_header.sh](/Volumes/SSD/archive/gov/governance-layer/tests/test_mcp_remote_public_host_header.sh)
  - reproduces the pre-fix host-header failure locally
  - verifies that an authenticated request with the configured public Host header now reaches the MCP transport instead of failing with `421`
- updated [mcp/README.md](/Volumes/SSD/archive/gov/governance-layer/mcp/README.md)
  - documents the truthful remote host-admission behavior when `GOVMCP_PUBLIC_BASE_URL` is configured

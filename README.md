# Governance Spine Starter (v0.1)

This folder is a starter "artifact spine" for the Governance Layer + Cecil harness workflow.

## How to use (minimal)
1. Put this folder in your repo (or `/Users/gregkeeter/archive/...`).
2. Keep `ACTIVE-TASK.md` as the single source of truth for what's in progress.
3. Any session must end in one of:
   - a doc update (patch),
   - a new/updated test,
   - a code change,
   - a report artifact (pass/fail).

## Where to edit first
- `ACTIVE-TASK.md`
- `ROADMAP.md`
- `SCOPE.md`
- `THREAT-MODEL.md`
- `DECISION-RECORD.md`

## Suggested automation
See `scripts/` for simple helpers. They are intentionally small and auditable.
